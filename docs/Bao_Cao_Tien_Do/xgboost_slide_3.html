NẾU LÀM CHI TIẾT XGBOOST CẦN CÁC BƯỚC GÌ?

Implementation Plan

Quy trình biến dữ liệu cảm biến thủy canh thành mô hình dự báo sinh trưởng chạy được trên Raspberry Pi

6 bước triển khai từ dữ liệu thật đến Web UI

1

Thu thập dữ liệu

ESP32-S3 gửi thông số môi trường qua MQTT. FastAPI nhận dữ liệu và ghi liên tục vào InfluxDB để tạo lịch sử theo thời gian.

sensor_data → InfluxDB
2

Làm sạch dữ liệu

Loại bỏ giá trị lỗi, thiếu, nhiễu bất thường. Ví dụ pH ngoài ngưỡng thực tế hoặc TDS bị nhảy đột ngột do cảm biến chưa ổn định.

remove null / outlier
3

Tạo đặc trưng

Tổng hợp dữ liệu theo giờ/ngày: nhiệt độ trung bình, ánh sáng tích lũy, pH/TDS trung bình, độ dao động môi trường và tuổi cây.

feature engineering
4

Tạo nhãn mục tiêu

Gán nhãn sinh trưởng dựa trên ngày trồng, ngày thu hoạch dự kiến hoặc số liệu đo thực tế như chiều cao, số lá, khối lượng cây.

growth_percent, days_remaining
5

Huấn luyện model

Dùng XGBoost Regression để học quan hệ giữa môi trường và tốc độ lớn của cây. Sau đó đánh giá sai số bằng MAE/RMSE.

XGBRegressor.fit()
6

Triển khai lên hệ thống

Lưu model thành file, đưa vào FastAPI, tạo endpoint dự báo. Web UI gọi API để hiển thị phần trăm sinh trưởng và ngày còn lại.

/api/ai/growth-predict

Ý chính khi báo cáo

Phần XGBoost không phải điều khiển bơm/quạt trực tiếp, mà là tầng dự báo thông minh. Nó giúp người dùng biết cây đang phát triển tốt hay chậm so với điều kiện môi trường đã ghi nhận.

Công thức hiểu đơn giản

Dữ liệu môi trường + Tuổi cây
→ XGBoost
→ % sinh trưởng + ngày còn lại

Sản phẩm cuối cần có

Bộ dữ liệu đã xử lý từ InfluxDB hoặc file CSV.
File mô hình đã huấn luyện, ví dụ growth_xgboost.pkl.
Endpoint FastAPI trả JSON gồm growth_percent và days_remaining.
Web UI hiển thị thanh tiến độ sinh trưởng và dự kiến ngày thu hoạch.
